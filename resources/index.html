BRACE Resources

The agent-security ecosystem, mapped to BRACE

BRACE composes with — it doesn't replace the work below. Use the standards to know what can go wrong, the reference implementations to see one way to build a control, and the open-source building blocks to actually build it. Each entry is tagged with the BRACE concern it serves.

How to read this. Tags show the BRACE concern an item most relates to: Build-time Run-time Agent Config Ecosystem Observability Governance. Status flags: Open source Commercial Free Emerging Deprecating.

Curated and verified mid-2026. Listing is not endorsement, and this space moves fast — verify status before you rely on anything. Know something we should add? Suggest a resource.

Open-source building blocks

The primitives you assemble BRACE-level controls from.

Identity & attribution → Agent

SPIFFE / SPIRE

CNCF-graduated workload identity — short-lived cryptographic SVIDs so agents authenticate without long-lived secrets. The foundation an agent identity chains from.

AgentObservabilityOpen source

Agent identity standards (IETF / NIST / OAP)

The 2025–2026 frontier of agent auth: an IETF agent-auth draft, NIST's agent-identity concept paper, and the Open Agent Passport spec. Worth tracking; none ratified yet.

AgentFreeEmerging

Authorization → Config / Agent / Ecosystem

Cedar

AWS's formally-verified, declarative authorization language/engine — the policy layer behind AgentCore Policy, usable standalone for capability-scoped tool gating.

OpenFGA

CNCF relationship-based (Zanzibar-style) authorization — well suited to agent delegation chains and "who can act on whose behalf."

AgentConfigOpen source

Open Policy Agent (OPA)

The CNCF-graduated general-purpose policy engine (Rego) for decoupled, context-aware authorization across gateways and tool calls.
